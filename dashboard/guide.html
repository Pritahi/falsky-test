Setup Guide

1 Quick Start

Get Falsky running in your CI pipeline in under 2 minutes.

1

Get Your API Key

Create an account at your Falsky instance to receive an API key.

2

Add the GitHub Action

Add this step after your test runner. Tests must output JUnit XML format.

YAML
# .github/workflows/ci.yml
name: CI with Falsky
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm test -- --reporter junit --reporter-options output=test-results.xml
      - uses: Pritahi/falsky-action@v1
        with:
          api_key: ${{ secrets.FALSKY_API_KEY }}
          junit_path: 'test-results/*.xml'

2 REST API

Write operations require the X-Falsky-API-Key header. Read operations are open.

MethodPathDescription
POST/api/junitIngest JUnit XML
POST/api/runsCreate test run from raw results
GET/api/dashboard?repo_name=XAggregated dashboard data

GET/api/runs?repo_name=XRecent CI runs
GET/api/reposAll tracked repos
GET/badge/{repo_name}SVG trust badge
POST/api/alerts/config?repo_name=XConfigure webhook alerts

cURL
curl -X POST https://your-falsky-instance.com/api/junit \
  -H "Content-Type: application/json" \
  -H "X-Falsky-API-Key: your-key" \
  -d '{"xml_content": "...", "repo_name": "my-repo"}'

3 Trust Score Algorithm

Each test gets a score from 0-100 based on four weighted components plus an optional flaky penalty.

Pass Rate

Bayesian pass rate with Beta-Binomial conjugate prior. Primary reliability signal.

Recent Trend

Compares last 5 runs vs previous 5. Improving tests get a bonus.

Duration Stability

Z-score analysis on execution times. Consistent = stable.

Error Consistency

Consistent errors = real bug. Varied errors = flaky.

Flaky Penalty (−5 to −20)

Additional penalty when flaky category is detected.

Flaky Categories

Timing

High duration variance — race conditions or timeout sensitivity.

Order Dependency

Outcome changes based on execution order.

Shared State

Shares mutable state with other tests.

Non-deterministic

Unpredictable external data or random values.
