01 The catch
STEP 01

Watch

Every run is recorded — result, retry count and timing for each test. Falsky's GitHub Action reads your JUnit XML output automatically.

STEP 02

Weigh

A Bayesian trust score weighs pass-rate, retries and timing variance to separate flaky from genuinely broken. Better than naive pass/total.

STEP 03

Quarantine

Flaky tests leave the merge path with an issue attached. Real bugs stay blocking. Your pipeline stays green.
