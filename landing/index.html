What Falsky Detects

⏱️

Timing Flakes

Tests that fail due to timeouts, slow responses, or race conditions. Detected via z-score anomaly detection on duration data.

🔗

Order Dependency

Tests that pass or fail depending on execution order. Sequential pattern analysis catches alternating and periodic failures.

🤝

Shared State

Tests that contaminate each other through shared databases, files, or services. Clustered failure patterns reveal the culprit.

🎲

Non-Deterministic Data

Tests that fail randomly due to unstable test data, UUIDs, or timestamps. High error diversity is the signal.

🌐

Environment Specific

Tests that pass locally but fail in CI (or vice versa). Cross-environment pass rate comparison catches these.

📊

Bayesian Scoring

Every test gets a trust score (0-100) using Beta-Binomial conjugate priors. Better than naive pass/total, especially with small samples.

How It Works

1

Connect your repo

Sign in, connect your GitHub repository. Falsky automatically detects your test framework.

2

Run your tests

Install the GitHub Action in your workflow. It reads JUnit XML output and sends it to Falsky's Trust Engine.

3

Get trust scores

Every test gets a Bayesian trust score. Flaky tests are categorized and flagged with confidence levels.

4

Fix before it breaks

See trends, catch degrading tests early, and quarantine unreliable tests before they block your pipeline.
